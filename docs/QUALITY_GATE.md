# Xenobot Quality Gate (English)

This document defines the practical quality gate used before block-level sync/publish.

For end-to-end operator flow (mode selection + smoke + gate + incident handling), see:
- `docs/OPERATIONS_RUNBOOK.md`

## 1) Fast gate (daily local)

```bash
cd Xenobot
scripts/xb quality gate --skip-platform --messages 600
```

Includes:
- strict workspace compile (`RUSTFLAGS='-D warnings' cargo check --workspace --offline`)
- docs consistency check (`scripts/check_docs_consistency.sh`)
- source hygiene pre-clean (`scripts/repo_hygiene.sh --apply`, removes transient `.DS_Store` artifacts)
- source hygiene check (`scripts/repo_hygiene.sh --strict-source`, blocks persistent `.DS_Store` leak in source tree)
- strict CLI suite
- strict API suite
- strict MCP suite (`cargo test -p xenobot-mcp --offline`)
- in-process API smoke contract (`api smoke`: health + SQL generate + SQL execute)
- perf baseline generation + perf contract check

## 2) Full gate (block closure)

```bash
cd Xenobot
scripts/xb quality gate --messages 1200
```

Includes everything in fast gate, plus:
- full 17-platform coverage suite
- MCP contract suite through coverage script

## 3) Perf gate tuning

You can override thresholds:

```bash
scripts/xb quality gate \
  --skip-platform \
  --skip-smoke \
  --messages 800 \
  --max-db-create-ms 30000 \
  --max-import-ms 120000 \
  --max-merge-import-ms 120000 \
  --max-query-ms 15000 \
  --max-total-ms 180000
```

Optional GPU constraints:

```bash
scripts/xb quality gate --skip-platform --require-gpu --max-gpu-ms 50
```

Temporary isolation flags:
- `--skip-mcp` (skip strict MCP suite)
- `--skip-smoke` (skip in-process API smoke contract)

## 4) Artifacts

- baseline report JSON:
  - default: `/tmp/xenobot-quality-gate-perf.json`
  - override via `--perf-output`
- additional perf logs are generated by baseline script under `reports/perf/` when using `xb perf baseline` directly.

## 5) Failure handling

- Any strict compile/test failure returns non-zero immediately.
- Source hygiene check fails when source tree still contains `.DS_Store` artifacts after pre-clean.
- API smoke contract failure returns non-zero with exact failing route and response details.
- Any perf threshold breach returns non-zero with explicit gate message.
- If frontend DNS is unavailable (`registry.npmjs.org`), Rust quality gate remains fully available because it runs offline and does not depend on frontend install.
